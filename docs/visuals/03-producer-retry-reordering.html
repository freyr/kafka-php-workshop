When retries reorder your records

two records, one transient error — and the partition log ends up backwards. why enable.idempotence=true (or max.in.flight=1) is the cure

PHP userland — two records, same key order-7 → same partition
producev(A: created)
producev(B: updated)
librdkafka queue: [A][B]
producer.simple — librdkafka defaults: retries on, many requests in flight
req 1: A — in flight
req 2: B — in flight
(right behind it)
✖ req 1 fails
(transient: timeout)
↻ req 1 retried: A
broker log · offset 41: B (updated)
offset 42: A (created)
fix 1 · producer.idempotent — enable.idempotence=true (PID + sequence numbers)
req 1: A (seq 0)
req 2: B (seq 1)
✖ req 1 fails
broker: want seq 0, got 1
→ reject B
↻ resend in order: A, B
offset 41: A (created)
offset 42: B (updated)
fix 2 · max.in.flight=1 — one request at a time (no pipelining)
send A → wait for ack ✓
only then: send B → ack ✓
log: A, B — ordered,
but every send is serialized
